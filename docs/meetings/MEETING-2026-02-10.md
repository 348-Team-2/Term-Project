# 4. Organizational Structure & Responsibilities

### Core Role Definitions

- **Analyst:** Defines system requirements and behavior.
- **Designer:** Plans architecture, UML, component structure.
- **Implementer:** Writes and maintains code.
- **Integrator:** Combines components into working system.
- **Tester:** Verifies functionality and correctness.
- **Change Control Manager:** Manages and approves change requests.
- **Configuration Manager:** Manages branches, versions, releases.
- **Project Manager:** Oversees planning, deadlines, coordination.
- **Reviewer (QA):** Reviews code and ensures quality.
- **Stakeholders:** Course staff and project clients.

# 5. Scrum / Workflow Structure

## Weekly Meeting Format

Each member reports:

1. What was completed?
2. What will I complete?
3. Are there any blockers?

---

## Development Flow

Scrum → Analyst → Designer → Implementer → Tester → Integrator

---

## Issue & PR Workflow

### 1. Analyst
- Creates issue in backlog:
  - Title
  - Description
  - Acceptance criteria
- Assigns to Designer

### 2. Designer
- Adds design details:
  - UML diagrams
  - Architecture decisions
  - Component structure
- Removes assignment
- Moves issue to **Ready**

### 3. Implementer
- Assigns issue to self
- Moves to **In Progress**
- Creates branch linked to issue
- Writes code
- Creates Pull Request
- Moves issue to **In Testing**

### 4. Tester
- Assigns to self
- Writes and executes test cases
- Creates bug issues if needed
- Moves issue to **In Review**

### 5. Integrator / Reviewer
- Reviews PR
- Merges into main branch
- Verifies build passes
- Confirms integration
